### Примеры промтов:

Ты — опытный инженер по тестированию. Сгенерируй набор тест-кейсов для интернет-магазина в формате таблицы Markdown со следующими колонками: ID, Название теста, Приор. (High/Medium/Low), Предусловия, Шаги, Ожидаемый результат, Постусловия. Тест-кейсы должны охватывать основной функционал: просмотр каталога и карточки товара, добавление и удаление товаров из корзины, изменение количества товаров, оформление заказа (позитивные и негативные сценарии, включая валидацию полей и спецсимволы), работу навигации (логотип, возврат на главную), адаптивность на мобильных устройствах и планшетах, проверку консоли браузера на ошибки при основных действиях, а также синхронизацию счётчика корзины между страницами. Каждый тест-кейс должен быть автономным, с чёткими шагами и ожидаемыми результатами. Используй реалистичные данные (например, названия товаров, поля формы). Постусловия указывай только при необходимости, иначе ставь прочерк. Результат оформи в виде единой таблицы Markdown.

Составь тест-план для интернет-магазина (демо-приложение на Polymer) в формате Markdown. Включи разделы: Введение (цель, объект, scope/out of scope), Приоритеты функциональности (таблица), Подход с уровнями тестирования (smoke, функциональное, UI/UX, интеграционное, кроссбраузерное, исследовательское), High-level сценарии, Технические особенности (Shadow DOM, адаптивность), Риски (таблица), Критерии начала/завершения, Метрики качества, Инструменты (ручные + автоматизация с обоснованием), Артефакты, Рекомендации. Оформи профессионально, лаконично, с нумерацией разделов. Начни с заголовка "# Тест-план для Polymer Shop".

Создай класс BasePage на TypeScript для Playwright с конструктором, принимающим page: Page, и геттерами для локаторов: логотип, пункты меню (мужская/женская верхняя одежда и футболки), список товаров, корзина и счётчик корзины, где все локаторы строятся через page.locator(), а также методами: open для перехода на главную страницу, clickLogo, clickRandomCategory для выбора случайной категории из меню с использованием утилиты getRandomItem, openCart с ожиданием загрузки сети и прокруткой вверх, getProductCardsCount для получения количества товаров на странице, clickOnRandomProduct для выбора случайного индекса через getRandomInt и клика по ссылке внутри карточки, добавив проверку наличия товаров перед кликом.
