# Тест-план для Polymer Shop  
URL: https://shop.polymer-project.org/

---

## 1. Введение

### 1.1 Цель тестирования
Обеспечить качество ключевых пользовательских сценариев интернет-магазина, выявить критические дефекты и подтвердить готовность приложения к использованию.

Основной фокус:
- корректность работы пользовательского пути (User Journey)
- стабильность основных функций
- отсутствие блокирующих дефектов в процессе покупки

---

### 1.2 Объект тестирования
Демо-приложение интернет-магазина, реализованное с использованием Web Components (Polymer).

---

### 1.3 Область тестирования (In Scope)
- Каталог товаров  
- Карточка товара  
- Корзина  
- Оформление заказа (Checkout)  
- Навигация и UI  

---

### 1.4 Вне области тестирования (Out of Scope)
- Бэкенд и API (отсутствуют)
- Сохранение данных между сессиями
- Платёжные интеграции
- Безопасность (Security testing)

---

## 2. Приоритеты функциональности

| Область | Описание | Приоритет | Обоснование |
|--------|----------|----------|------------|
| Каталог товаров | Просмотр списка товаров и переход в карточку | High | Точка входа пользователя — влияет на конверсию |
| Карточка товара | Детали товара + добавление в корзину | High | Ключевой этап принятия решения |
| Корзина | Управление товарами | High | Потенциальная потеря заказа при ошибке |
| Checkout | Завершение заказа | Medium | В демо-версии упрощён, но важен для сценария |
| Навигация / UI | Меню, переходы | Low | Не блокирует покупку, но влияет на UX |

---

## 3. Подход к тестированию

### 3.1 Уровни тестирования

#### 1. Smoke testing
Проверка критического пути:
- Открытие главной страницы
- Загрузка каталога
- Переход в карточку
- Добавление в корзину

---

#### 2. Функциональное тестирование

Позитивные сценарии:
- Добавление товара в корзину
- Изменение количества
- Удаление товара
- Успешный checkout

Негативные сценарии:
- Пустые поля в checkout
- Некорректный ввод (спецсимволы)
- Попытка оформить заказ без товаров

---

#### 3. UI/UX тестирование
- Проверка адаптивности (mobile / tablet / desktop)
- Отсутствие визуальных дефектов:
  - наложения элементов
  - обрезанный текст
- Проверка кликабельности элементов

---

#### 4. Интеграционное тестирование (frontend)
- Синхронизация:
  - добавление товара → обновление корзины
  - изменение количества → пересчёт

---

#### 5. Кроссбраузерное тестирование
Минимальный набор:
- Chrome
- Firefox

---

#### 6. Исследовательское тестирование
- Свободный сценарий использования
- Поиск нестандартных дефектов

---

## 4. Тестовые сценарии (High-level)

### 4.1 Основной пользовательский путь
1. Открыть сайт  
2. Выбрать товар  
3. Перейти в карточку  
4. Добавить в корзину  
5. Перейти в корзину  
6. Изменить количество  
7. Перейти в checkout  
8. Завершить заказ  

Полное описание тестовых сценариев с шагами, тестовыми данными и ожидаемыми результатами приведено в отдельном файле — test-cases.md.
---

## 5. Технические особенности

### 5.1 Shadow DOM
- Требует специальных селекторов
- Использовать возможности Playwright

---

### 5.2 Frontend-only приложение
- Данные не сохраняются
- Тестирование ограничено одной сессией

---

### 5.3 Адаптивность
- Проверка на:
  - 320px (mobile)
  - 768px (tablet)
  - 1280px+ (desktop)

---

## 6. Риски

| Риск | Влияние | Вероятность | Митигация |
|-----|--------|------------|----------|
| Некорректная работа Shadow DOM | Высокое | Средняя | Использование Playwright |
| Визуальные баги на мобильных | Среднее | Высокая | UI тестирование |
| Нестабильность демо-данных | Низкое | Средняя | Перезапуск сессии |

---

## 7. Критерии начала и завершения

### 7.1 Entry criteria
- Сайт доступен
- Тестовая среда настроена
- Браузеры установлены

---

### 7.2 Exit criteria
- Выполнены все тест-кейсы
- Нет критических (Blocker / Critical) багов
- Автотесты проходят стабильно
- Подготовлен отчёт о тестировании

---

## 8. Метрики качества

- Количество найденных дефектов
- Количество критических багов
- Pass rate тестов (%)
- Покрытие ключевых сценариев
